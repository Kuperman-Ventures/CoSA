Release Flow (Every Time)

1. Merge or push latest code to main branch.
2. Confirm Vercel production deploy is complete and healthy.
3. Open production URL in a fresh browser tab.
4. Run all checks below, in order.

A. Production Smoke Test

  • Production URL loads with no blank screen.
  • Bottom navigation is visible and clickable.
  • No obvious layout breaks on your main device.

B. Auth (Google SSO)

  • Signed-out state shows Google sign-in action.
  • Google login succeeds and returns to production app.
  • Signed-in email appears in header.
  • Sign out works and returns to signed-out state.

C. Phase 1 - Today + Timer

Timer States

  • Not Started is visible before start.
  • Running starts countdown correctly.
  • Pause works and Resume returns to Running.
  • Complete changes state to Completed.
  • Cancel changes state to Cancelled.
  • Overrun appears when timer hits zero and counts upward.

Prompt Rules

  • Complete is blocked without "What was actually completed?" text.
  • Done + Outcome task requires Yes/No outcome before completion.
  • Overrun message appears clearly in red state.

Ventures / Encore Rule

  • Start is blocked if definition of done has fewer than 10 words.
  • Start works once definition has 10+ words.
  • Completion recap shows definition vs actual completed.

Timer Metrics

  • Pause count increments correctly.
  • Pause duration accumulates correctly.
  • Cancelled time displays remaining time when cancelled.
  • Overrun and time saved values update correctly.

D. Phase 2 - Task Library

Fields + Editability

  • Can edit Name, Track, Subtasks, Default Time, Frequency, Completion Type, KPI Mapping, Time Block, Status.
  • Add Task creates a new editable library task.
  • Validation shows clear issues for required missing fields.

Status Behavior

  • Active tasks are marked deployable.
  • Paused tasks are excluded from deploy.
  • Archived tasks are excluded from deploy.
  • All/Active/Paused/Archived filter works.

Deploy Snapshot Rule (Most Important)

  • Deploy preview shows Ready vs Blocked Active tasks.
  • Deploy is blocked if any Active task has validation errors.
  • Deploy succeeds once Active tasks are valid.
  • Last deployment timestamp updates.
  • Today queue reflects the new snapshot after deploy.

Non-Retroactive Proof Test

  1. Deploy to Today.
  2. Edit a library task (name or estimate).
  3. Do not deploy again yet.
  • Today still shows old values (unchanged).
  1. Deploy again.
  • Today now shows updated values from library.

E. Cross-Device Production Check

  • Login on desktop/laptop browser works.
  • Open production URL on second device and login with same Google account.
  • UI is readable and usable on mobile browser.
