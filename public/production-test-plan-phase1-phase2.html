CoSA Production Test Plan (Phase 1 + 2)

Production-first runbook. Build -> push -> deploy -> test on production URL.

Rule: Do not block on local testing first.

Run Info

Date:
Tester:
Production URL:

Release Flow

1. Push latest code to main.
2. Wait for Vercel production deploy complete.
3. Open production URL in a fresh browser tab.
4. Execute checks below in order.

A. Production Smoke

  • App loads with no blank/error page.
  • Bottom nav is visible and works.
  • Main layout is usable on your current device.

B. Auth (Google SSO)

  • Signed-out state shows Google sign-in.
  • Google login succeeds and returns to app.
  • Signed-in email appears in header.
  • Sign out works.

C. Phase 1 (Today + Timer)

State coverage

  • Not Started
  • Running
  • Paused
  • Completed
  • Cancelled
  • Overrun

Rules

  • Done + Outcome requires Yes/No outcome.
  • Complete blocked without completion notes.
  • Overrun prompt appears at zero and timer counts up.
  • Ventures start blocked without 10-word definition of done.

D. Phase 2 (Task Library)

  • All 9 fields editable.
  • Status behavior works: Active deploys; Paused/Archived excluded.
  • Deploy preview shows Ready vs Blocked.
  • Deploy blocked when Active task has validation errors.
  • Last deployment timestamp updates on deploy.

Non-retroactive proof

  1. Deploy library to Today.
  2. Edit a library task.
  3. Do not deploy again.
  • Today remains unchanged.
  1. Deploy again.
  • Today updates to new snapshot values.

Sign-Off

  • Phase 1 accepted in production.
  • Phase 2 accepted in production.
